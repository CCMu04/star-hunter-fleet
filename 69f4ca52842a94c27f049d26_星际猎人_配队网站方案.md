## 一、项目概述

### 1.1 项目背景

《星际猎人》（原《无尽的拉格朗日》）是一款太空科幻SLG手游，目前已有199艘已实装舰船、9大势力、多种舰船类型与模块系统。当前配队工具以微信小程序"配舰星港"（火星工坊开发）为主，但小程序存在以下局限：

- **功能单一**：仅支持配队模拟，缺少图鉴浏览、蓝图加点、模块搭配等一体化功能
- **数据更新慢**：依赖手动维护，新舰船/版本更新后数据滞后
- **交互受限**：小程序界面空间有限，无法展示完整的舰船数据和配队分析
- **无社区功能**：玩家无法分享配队方案、讨论策略

### 1.2 项目目标

打造一个**一站式网页版配队平台**，整合以下核心能力：

1. **舰船图鉴**：199艘舰船完整数据（火力、资源、模块、评级等）
2. **智能配队**：基于指挥值、站位、武器类型的舰队编组工具
3. **蓝图加点**：可视化蓝图强化树模拟器
4. **模块搭配**：超主力舰模块系统配置工具
5. **阵容分享**：玩家社区配队方案分享与讨论
6. **数据同步**：自动从Wiki抓取最新数据

### 1.3 目标用户

| 用户类型 | 核心需求 |
|---------|---------|
| 新手玩家 | 快速了解舰船属性、获取推荐配队方案 |
| 中阶玩家 | 蓝图加点模拟、模块搭配优化 |
| 高阶玩家 | PVP配队策略分析、阵容克制计算 |
| 攻略作者 | 创建/分享配队方案、编写攻略 |

### 2.3 智能配队模块

#### 2.3.1 功能描述

核心功能模块，支持玩家自由编组舰队，并提供智能推荐。

#### 2.3.2 配队规则引擎

配队需遵循游戏内的核心规则：

| 规则 | 说明 |
|------|------|
| **指挥值上限** | 根据基地等级不同，舰队总指挥值上限不同（如7本=300、9本=400） |
| **站位分配** | 前排（承伤）、中排（综合）、后排（输出/支援） |
| **编队上限** | 每种舰船有服役上限（如战巡最多6艘） |
| **舰载机编队** | 战机/护航艇以编队为单位（如2架一组），需由载机平台搭载 |
| **旗舰技** | 部分舰船拥有旗舰技，影响全队（如"舰队集火"） |

#### 2.3.4 智能推荐逻辑

参考游戏内已上线的"阵容智能推荐机制"，结合玩家实际情况推荐：

```
输入：
  - 玩家基地等级 → 确定指挥值上限
  - 玩家持有蓝图 → 过滤可用舰船池
  - 用途选择（PVP/PVE/攻城/综合）→ 确定配队策略

推荐流程：
  1. 根据基地等级确定配队框架（前排/中排/后排比例）
  2. 从可用舰船池中筛选各位置最优选
  3. 计算舰队综合属性（火力、生存、防空等）
  4. 若缺少关键舰船，提供替代建议
  5. 生成阵容分析说明
```

#### 2.3.5 舰队属性计算

```javascript
// 舰队综合属性计算示例
function calculateFleetStats(fleet) {
  return {
    totalAntiShip: sum(ship.stats.antiShipFirepower * ship.count),
    totalAntiAir: sum(ship.stats.antiAirFirepower * ship.count),
    totalSiege: sum(ship.stats.siegeFirepower * ship.count),
    totalStructure: sum(ship.stats.structureValue * ship.count),
    totalCommandValue: sum(ship.commandValue * ship.count),
    kineticRatio: kineticDamage / totalDamage,    // 实弹占比
    energyRatio: energyDamage / totalDamage,      // 能量占比
    antiAirRating: calculateAntiAirRating(fleet),  // 防空评级
    frontlineSurvival: calculateSurvival(fleet),   // 前排生存能力
    flagshipSkills: getActiveFlagshipSkills(fleet), // 激活的旗舰技
  }
}
```

### 2.4 蓝图加点模块

#### 2.4.1 功能描述

模拟游戏内的蓝图强化树系统，帮助玩家规划加点方案。

#### 2.4.2 强化树结构

```
舰船蓝图
├── 武器系统（主系统）
│   ├── 强化项A → 子节点A1, A2
│   └── 强化项B → 子节点B1
├── 装甲系统
│   ├── 强化项C → 子节点C1
│   └── 强化项D
├── 动力系统
│   └── 强化项E → E1, E2
├── 指挥系统
│   └── 强化项F
└── [超主力舰专属]
    ├── M系统（模块系统）
    │   ├── M1 → M2 → M3
    ├── A系统（武器模块）
    │   ├── A1, A2
    ├── B系统（搭载模块）
    │   ├── B1, B2
    └── C系统（功能模块）
        ├── C1, C2, C3
```

#### 2.4.3 交互设计

- **可视化树状图**：用Canvas/SVG绘制强化树，节点可点击展开/折叠
- **加点模拟**：点击节点分配加点，实时计算属性变化
- **加点上限**：根据游戏规则限制总加点数
- **方案保存/分享**：保存加点方案，生成分享链接
- **推荐加点**：为每艘舰船提供社区推荐的加点方案

### 2.5 模块搭配模块

#### 2.5.1 功能描述

针对战列巡洋舰、战列舰、航空母舰等超主力舰的模块系统配置工具。

#### 2.5.2 模块系统说明

超主力舰拥有M/A/B/C四类模块插槽：

| 模块类型 | 说明 | 示例 |
|---------|------|------|
| M系统 | 核心模块，定义舰船核心定位 | "磐石"I/II/III型防御系统 |
| A系统 | 主武器模块 | 导弹发射系统、能量投射武器系统 |
| B系统 | 搭载/功能模块 | 护航艇搭载系统、投射矩阵系统 |
| C系统 | 辅助功能模块 | 防空导弹系统、维修无人机系统 |

#### 2.5.3 交互设计

- **模块槽位可视化**：展示舰船的M/A/B/C模块槽位
- **模块库浏览**：浏览所有可用模块及其属性加成
- **拖拽搭配**：拖拽模块到槽位进行搭配
- **属性预览**：实时预览模块搭配后的属性变化
- **方案对比**：对比不同模块搭配方案的属性差异

#### 2.6.1 功能描述

#### 2.6.2 分析维度

```
克制分析

│   ├── 实弹 vs 能量护盾
│   ├── 能量 vs 实弹装甲
│   └── 混合伤害的平衡性
├── 站位克制
│   ├── 前排突破能力（能否击穿对方前排）
│   ├── 后排输出效率（能否有效打击对方后排）
│   └── 防空覆盖（能否拦截对方舰载机）
├── 速度克制
│   ├── 速度差分析（谁先手）
│   └── 突防/巡航速度对比
└── 综合评分
    └── 胜率预估（基于属性对比的简化模型）
```

## 三、技术方案

### 3.1 技术栈选型

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| **前端框架** | React 18 + TypeScript | 组件化开发，生态丰富 |
| **UI组件库** | Ant Design 5 / Tailwind CSS | 响应式设计，暗色主题支持 |
| **状态管理** | Zustand | 轻量级，适合中等复杂度应用 |
| **可视化** | D3.js / ECharts | 强化树、雷达图、统计图表 |
| **后端** | Node.js + Express / Nuxt.js SSR | SEO友好，首屏加载快 |
| **数据库** | PostgreSQL + Redis | 结构化数据+缓存 |
| **图片存储** | OSS / Cloudflare R2 | 舰船图片CDN加速 |
| **部署** | Vercel / Cloudflare Pages | 免费额度，全球CDN |

### 3.3 数据同步方案

```
Wiki页面 → 爬虫脚本 → 数据清洗 → JSON文件 → 前端加载
                                    ↓
                              PostgreSQL数据库
                                    ↓
                              社区数据（方案/评论）
```

- **初始数据**：使用已爬取的199艘舰船JSON数据
- **增量更新**：定时爬虫（每周一次）检查Wiki更新
- **版本管理**：每次数据更新记录版本号，前端可对比版本差异

### 3.4 性能优化

| 优化点 | 方案 |
|--------|------|
| 首屏加载 | SSR + 静态数据预渲染 |
| 图片加载 | WebP格式 + 懒加载 + CDN |
| 大列表渲染 | 虚拟滚动（react-window） |
| 数据缓存 | Service Worker + IndexedDB |
| API请求 | 数据静态化，减少服务端压力 |

3. **克制分析**：独创的配队克制分析功能
4. **开放平台**：网页版全平台访问，SEO友好
5. **社区驱动**：玩家贡献配队方案，形成数据飞轮

## 八、风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|---------|
| Wiki数据结构变更 | 数据解析失败 | 监控爬虫异常，设置告警 |
| 版权/合规风险 | 收到DMCA通知 | 仅展示数据，不复制原创内容；注明数据来源 |
| 用户量不足 | 社区活跃度低 | 先通过SEO获取搜索流量，再逐步建立社区 |
| 技术复杂度高 | 开发周期延长 | MVP优先，分阶段迭代 |
| 游戏版本更新 | 数据频繁变动 | 建立数据版本管理，支持快速更新 |

## 九、总结

本方案旨在打造一个以**数据驱动**为核心的星际猎人配队平台，通过整合199艘舰船的完整数据、智能配队推荐、蓝图加点模拟、模块搭配工具和社区分享功能，为玩家提供远超现有小程序的一站式配队体验。项目采用React+Node.js技术栈，分三个阶段迭代开发，预计8周完成核心功能上线。
